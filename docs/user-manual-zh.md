# CapyMeet 使用手册

本文档介绍 CapyMeet 的日常使用方式，适用于管理员、普通预约用户和会议室门口 Pad 使用场景。

## 1. 系统简介

CapyMeet 用于管理会议室预约、门口 Pad 状态展示、管理员后台配置、审批流程、预约链接和可选邮件通知。系统独立管理会议数据，不依赖 Google Calendar、Outlook 或其他外部日历。

主要入口：

- 公共预约页 `/`：用户无需登录即可查看会议室并提交预约。
- 预约链接 `/book/<token>`：管理员生成的公开预约入口，可以是全局链接或房间专属链接。
- Pad 门口屏 `/pad/<deviceCode>`：展示会议室当前状态、今日日程和扫码预约二维码。
- 管理后台 `/admin/login`：管理员登录后管理会议室、设备、预约、审批、链接、邮件和账号。
- 取消提示页 `/cancel`：当前版本不提供自助取消，用户需要联系管理员处理。

## 2. 用户预约会议

用户打开公共预约页或扫码进入预约链接后，可以按以下流程提交预约：

1. 选择会议室。
2. 选择预约日期、开始时间和结束时间。
3. 查看会议室规则、开放时间和已有预约。
4. 填写会议主题、联系人姓名和邮箱。
5. 提交预约。

系统会检查会议室是否启用、预约时间是否在开放范围内、预约时长是否满足规则，以及是否与已有已确认或待审批预约冲突。会议室如开启审批，预约会进入待审批状态；否则会直接确认。

## 3. 修改或取消预约

当前版本已经移除自助取消入口。用户如需修改或取消预约，需要联系管理员在后台处理。

管理员可以在后台 Bookings 页面查看预约列表，并根据实际情况取消或删除预约。取消后的预约不会继续占用会议室时段。

## 4. Pad 门口屏使用

Pad 页面路径格式为：

```text
/pad/<deviceCode>
```

其中 `<deviceCode>` 是管理员在后台 Devices 中创建的设备码。

Pad 门口屏会显示：

- 当前绑定或选中的会议室。
- 房间当前是否空闲。
- 当前会议、主持人和时间段。
- 当天后续已确认预约。
- 指向当前房间预约页的二维码。

Pad 页面每 10 秒刷新时间和日程，并向后台上报设备心跳。待审批预约不会显示在门口日程中。设备码不存在、设备被禁用或网络异常时，页面会显示相应错误。

建议把 Pad 固定在会议室门口，保持稳定 Wi-Fi 和持续供电，并在系统设置中关闭自动锁屏或启用 kiosk / 单应用模式。

## 5. 管理后台使用

管理后台入口：

```text
/admin/login
```

管理员输入邮箱和密码登录。登录成功后，系统会保存管理员令牌，并进入后台首页。

后台主要模块：

| 模块 | 用途 |
| --- | --- |
| Dashboard | 查看会议室当前状态、今日预约、待审批数量和后续会议 |
| Rooms | 创建、编辑、启用或禁用会议室，并配置开放时间、缓冲时间、预约时长和审批规则 |
| Devices | 创建和维护 Pad 设备，设置设备码、设备名称、默认会议室和启用状态 |
| Bookings | 查看、筛选、手动创建、取消或删除预约 |
| Approvals | 审核待审批预约，通过时会重新检查冲突 |
| Links & QR | 创建全局预约链接或房间专属预约链接，并查看二维码 |
| Email | 开关邮件通知，配置邮件主题、回复说明，并发送测试邮件 |
| Admins | 创建管理员账号，启用或禁用账号，修改姓名或重置密码 |

## 6. 预约链接和二维码

管理员进入 `/admin/links` 后，可以创建两类预约链接：

- 全局预约链接：用户进入后自行选择会议室。
- 房间专属链接：用户进入后默认预约指定会议室。

系统会为每个链接生成 URL 和二维码。二维码适合张贴在会议室门口、公司内网页或活动通知中。不再使用的链接建议禁用或删除。

## 7. 邮件通知规则

邮件通知是可选能力。

只有在以下条件同时满足时才会发送邮件：

- 预约里填写了邮箱。
- 系统已配置邮件服务参数。
- 后台 Email 设置中启用了通知。

支持的环境变量：

- `RESEND_API_KEY` 或 `EMAIL_API_KEY`
- `EMAIL_FROM`

邮件发送失败不会阻止预约、审批或取消操作。管理员可以在后台 Email 页面发送测试邮件，确认当前配置是否可用。

## 8. 部署和运行配置

系统面向 Cloudflare 部署，主要依赖：

- Cloudflare Pages：前端页面。
- Cloudflare Pages Functions：API。
- Cloudflare D1：数据库。

必需配置：

- `DB`：Cloudflare D1 绑定。
- `JWT_SECRET`：管理员登录令牌密钥，长度至少 32 个字符。
- `PUBLIC_BASE_URL`：系统线上访问地址，用于生成预约链接和二维码。

本地常用命令：

```bash
npm install
npm test
npm run build
cp wrangler.example.toml wrangler.toml
npm run db:migrate:local
```

`wrangler.toml` 应保存本地或部署环境的真实 D1 配置，不应提交到仓库。

## 9. 当前版本限制

- 不接入外部日历。
- 不支持短信验证码。
- 不提供用户自助取消或修改预约入口。
- 邮件发送依赖第三方邮件服务配置。
- 首个管理员账号需要部署后手动写入 D1。

## 10. 常见问题

### 为什么预约时提示冲突？

同一会议室中，只要已有 `confirmed` 或 `pending_approval` 状态的预约与当前选择时段重叠，系统就会拒绝新预约。会议室配置的缓冲时间也会纳入冲突检测。

### 预约必须填写邮箱吗？

当前预约表单需要填写邮箱。邮件通知只有在系统完成邮件配置并启用后才会发送。

### 为什么 Pad 上看不到刚提交的预约？

Pad 只展示已确认预约。待审批预约不会显示在门口日程中。已确认预约创建或修改后，Pad 最多需要等待下一次 10 秒刷新。

### 管理员被禁用后还能继续使用吗？

不能。后台接口会在每次访问受保护接口时重新检查管理员是否仍启用。

### 邮件发送失败会影响预约吗？

不会。邮件发送失败不会阻止预约、审批或取消状态变化。
